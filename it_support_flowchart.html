Описание процесса

Участники системы:
  • Сотрудник - создает заявки через Telegram
  • Telegram-бот - интерфейс системы
  • Техники - обрабатывают заявки
  • Админ - управляет системой
  • Группа ИТ - получает дубликаты заявок
Статусы заявок:
  • NEW - новая заявка
  • IN_PROGRESS - в работе
  • ON_HOLD - на паузе
  • RESOLVED - выполнена
  • CLOSED - закрыта с оценкой
Хранилище данных:
  • users.json - пользователи
  • tickets.json - заявки
  • settings.json - настройки
  • events.jsonl - журнал событий
Ключевые особенности:
  • Атомарное назначение техника
  • Возможность постановки на паузу
  • Обязательная оценка при закрытии
  • Полное логирование всех изменений
  • Уведомления всех участников
